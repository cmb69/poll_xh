Poll_XH - a CMSimple_XH Plugin

Poll_XH facilitates to place polls on your CMSimple_XH website. You can have as many polls you like, with as many options you like (single or multiple choice). The voters are distinguished by cookie and IP address, so cheating is somewhat unlikely.

Requirements

Poll_XH requires CMSimple_XH ≥ 1.6.3 with Pfw_XH ≥ 0.2.0, and PHP ≥ 5.4.0.

Installation

The installation is done as with many other CMSimple_XH plugins. See the CMSimple_XH wiki for further details.

  1. Backup the data on your server.
  2. Unzip the distribution on your computer.
  3. Upload the whole directory poll/ to your server into CMSimple_XH's plugins directory.
  4. Set write permissions to the subdirectories css/ and languages/.
  5. Switch to "Poll" in the back-end to check if all requirements are fulfilled.

Settings

The plugin's configuration is done as with many other CMSimple_XH plugins in the website's back-end. Select "Poll" from "Plugins".

You can change the default settings of Poll_XH in "Plugin Config". Hints for the options will be displayed when hovering over the help icon with your mouse.

Localization is done in "Plugin Language". You can translate the character strings to your own language, or customize them according to your needs.

The look of Poll_XH can be customized in "Plugin Stylesheet".

Usage

You can embed a poll on a CMSimple page with the following plugin call:

{{{PLUGIN:poll('name-of-the-poll');}}}

To embed a poll in the template use:

<?php echo poll('name-of-the-poll');?>

Instead of 'name-of-the-poll' you can use any name as long as it consists of lowercase letters (a-z), digits (0-9) and hyphens only. You can embed as many polls on any single page as you like – they're working independent of each other as long as they have different names.

If the poll has not ended and the visitor has not voted yet, he will be presented the voting view and can submit his vote. After he has voted, he can see the results of the poll.

Currently the poll data files have to be created and edited manually. Just put a file name-of-the-poll.csv to Poll_XH's data folder (plugins/poll/data/ by default). Each voting option has it's own line in the file. Additionally there are two meta options, namely %%%MAX%%% and %%%END%%%, which you have to put in separate lines too. Both are optional.

%%%MAX%%%→3

will make a multiple-choice poll where the user can check 3 options at most, and with

%%%END%%%→1335744000

you can specify the end date of the poll as UNIX timestamp. Note that the → stands for a <tab> character. To calculate the UNIX timestamp of a date, you can use an online converter.

As an example a cmsimple.csv file is delivered, which should explain the file format. The end date of this poll is set to December, 31st 2012. You can embed this poll on a page by writing:

{{{PLUGIN:poll('cmsimple');}}}
